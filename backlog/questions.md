> 最後更新：2026-07-08

## 關於書的本質

- [x] 這本書的目標長度？
  → **8–10 萬字**，每章約 8,000–10,000 字。夠深不拖，符合「直白不繞圈」的風格。
- [x] 出書方式？
  → **個人 repo 先行**，出版未來再說。不是公開出版專案，是自己的寫作倉庫。
- [x] 是否需要搭配網站或線上資源？
  → **不需要**。Repo 本身就是承載體，不用浪費力氣做網站。
- [x] 是否需要練習本或工作簿？
  → **要，整合在書稿內**。每章結尾放練習區塊（Markdown），不獨立出版。
- [x] 是否需要建立讀者社群？
  → **現階段不需要**。出版再說。

## 關於作者定位

- [x] 作者在書中的「人設」要維持什麼層次？
  → **走過深度迷茫、持續校準中的同行者**。不是「已經成功上岸的人」。
  → 書中大方承認「現在也還在過程中」——這反而是本書的誠信所在。
  → 語氣用「那時候的我……」、「後來我才理解……」、「現在回頭看……」。
- [x] 作者的真實身份要揭露多少？
  → 用 **Ryan** 作為書中自稱即可。
  → 經歷事實（輟學、做過什麼、賠過多少）保留真實。
  → 真實全名和現在職業，等到真要出版時再決定。
  → 他人隱私一律匿名。

## 關於框架

- [x] 「道術勢器」和「鬆開看清行動接住」兩個框架的關係要不要在書中明確定義？
  → **要。** 素材已定義，只需提煉到書中：
  - 道術勢器 = **結構診斷**（卡在哪個層面？）
  - 鬆開看清行動接住 = **校準循環**（怎麼從卡住到能動？）
  - `loosen-see-do-catch.md` 已標明對應關係：鬆開→道、看清→道+術+勢、行動→術、接住→器
- [x] 兩個框架是否要合併成一個統一的圖示或模型？
  → **要。** 一個視覺模型：外圈道術勢器（診斷維度），內圈鬆開→看清→行動→接住（校準循環）
- [x] 要不要有「本書框架速覽」之類的單頁指南？
  → **要。** 放在前言之後、第一章之前。包含圖示 + 10 章一句話摘要。
